Git es un software de control de versiones que nos permite registrar los cambios hechos a archivos de computadora. Un sistema de control de versiones registra los cambios a un archivo, o un conjunto de archivos a lo largo del tiempo, de manera que se puedan recuperar ciertas versiones posteriormente. Git hace esto a través de repositorios (almacenamiento virtual del proyecto).

GitHub es una plataforma web que permite almacenar los repositorios de una manera remota, con el fin de colaborar y coordinar los avances de un proyecto que pertenece a un equipo de trabajo.

Algunas de las ventajas de usar GitHub son:
  • Facilita la colaboración entre los miembros de un equipo de trabajo.
  • Permite llevar un registro de los cambios hechos a un proyecto.
  • Facilita la revisión de código.
  • Permite la integración continua.
  • Facilita la revisión de código.
El flujo de trabajo en GitHub es el siguiente:
  1. Se crea un repositorio en GitHub.
  2. Se clona el repositorio en la computadora local.
  3. Se confirman los cambios
  4. Se confirman los cambios
  5. Se envían los cambios al repositorio remoto.

El flujo de trabajo cuando eres parte de un equipo es asi:
  1. Se crea un repositorio en GitHub.
  2. Se clona el repositorio en la computadora local.
  3. Se crean ramas para trabajar en los cambios.
  4. Se hacen cambios en los archivos.
  5. Se confirman los cambios
  6. Se envían los cambios al repositorio remoto.
  7. Se hace un pull request para revisar los cambios.
  8. Se hace un merge para unir los cambios al repositorio principal.

Comandos principales

Algunos de los comandos principales de Git son:

Comando Descripcion
git add (documento) Este comando se utiliza para agregar archivos al área de preparación.
git commit - m (descripcion) Este comando se utiliza para confirmar los cambios y almacenarlos en el repositorio.
git push - u origin main Este comando se utiliza para enviar los cambios confirmados a un repositorio remoto.
git pull Este comando se utiliza para traer los cambios de un repositorio remoto a un repositorio local.
git clone (url) Este comando se utiliza para clonar un repositorio remoto en un repositorio local.
git status Este comando se utiliza para ver el estado de los archivos en el repositorio.
git restore (archivo) Este comando se utiliza para sacar un archivo del staged area.
git merge Este comando se utiliza para unir dos ramas.
git branch -a Este comando se utiliza para ver las ramas que existen en el repositorio.
git checkout -b (rama) Este comando se utiliza para crear una rama nueva y cambiar de la rama actual a la nueva.
git checkout (rama) Este comando se utiliza para cambiar de una rama a otra.
git switch Este comando tambien se utiliza para cambiar de una rama a otra.
git branch -d (rama) Este comando se utiliza para eliminar una rama.
git push origin -d (rama) Este comando se utiliza para eliminar una rama del repositorio remoto.
