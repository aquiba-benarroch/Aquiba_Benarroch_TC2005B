1. ¿Qué es el DOM?

Respuesta ChatGPT:

El DOM (Document Object Model) es una interfaz de programación para los documentos HTML y XML. Básicamente, representa la estructura de la página web como una jerarquía de nodos (etiquetas, texto, atributos, etc.). Gracias al DOM, los lenguajes de programación como JavaScript pueden acceder y manipular el contenido, la estructura y los estilos de la página de forma dinámica.

Respuesta Copilot:

El DOM (Document Object Model) es una representación estructurada del documento HTML o XML que permite a los lenguajes de programación, como JavaScript, interactuar con el contenido, estructura y estilo de una página web. Es una interfaz que convierte el documento en un árbol de nodos.

¿Es verdad? Sí, ambas respuestas describen acertadamente el DOM.

Fuente que valida: MDN - Document Object Model

2. ¿Qué es un nodo?

Respuesta ChatGPT:

Un nodo es la unidad más básica dentro del DOM. Cada elemento HTML, cada texto dentro de un elemento, cada atributo, cada comentario y hasta el propio documento en sí mismo están representados como nodos en la estructura jerárquica del DOM.

Respuesta Copilot:

Un nodo es la unidad básica del DOM. Puede representar un elemento HTML, un atributo, un texto, un comentario o incluso el propio documento. Por ejemplo, un <div> o un texto dentro de un párrafo son nodos.

¿Es verdad? Sí, ambas explicaciones son correctas.

Fuente que valida: MDN - Node

3. ¿Cómo se accede a un elemento del DOM?

Respuesta ChatGPT:

Para acceder a un elemento del DOM en JavaScript, se suelen usar los siguientes métodos:

  • document.getElementById("miId")
  • document.getElementsByClassName("miClase")
  • document.getElementsByTagName("p")
  • document.querySelector("selectorCSS")
  • document.querySelectorAll("selectorCSS")

Respuesta Copilot:

Puedes acceder a un elemento del DOM utilizando métodos como:

  • document.getElementById("id")
  • document.getElementsByClassName("class")
  • document.getElementsByTagName("tag")
  • document.querySelector("selector")
  • document.querySelectorAll("selector")

¿Es verdad? Sí, las listas de métodos coinciden y son correctas.

Fuente que valida: MDN - Document

4. ¿Cómo se modifica un elemento del DOM?

Respuesta ChatGPT:

Algunas maneras de modificar un elemento del DOM incluyen:

  • Modificar el contenido: element.innerHTML o element.textContent
  • Modificar atributos: element.setAttribute() o element.id = "nuevoID"
  • Modificar estilos: element.style.propiedadCSS = "valor"
  • Crear o eliminar elementos: document.createElement(), element.appendChild(), element.removeChild()

Respuesta Copilot:

Puedes modificar un elemento del DOM de varias maneras:

  • Cambiar su contenido: element.textContent = "Nuevo texto";
  • Cambiar su HTML interno: element.innerHTML = "<b>Nuevo contenido</b>";
  • Cambiar atributos: element.setAttribute("atributo", "valor");
  • Cambiar estilos: element.style.color = "red";

¿Es verdad? Sí, ambas descripciones coinciden.

Fuente que valida: MDN - Manipulación del DOM

5. ¿Cuál es la diferencia entre getElementBy* y querySelector*?

Respuesta ChatGPT:

  • getElementBy* (ej. getElementById, getElementsByClassName) retorna colecciones vivas o un único elemento, según busque por id, clase o etiqueta.
  • querySelector* (ej. querySelector, querySelectorAll) acepta cualquier selector CSS, retornando el primero o todos los elementos que coincidan. querySelectorAll retorna un NodeList estático.

Respuesta Copilot:

getElementBy* (como getElementById o getElementsByClassName) devuelve elementos basados en su ID, clase o etiqueta. Los métodos getElementsBy* devuelven colecciones HTML (listas vivas).
querySelector* utiliza selectores CSS para buscar elementos. querySelector devuelve el primer elemento que coincide, mientras que querySelectorAll devuelve una lista estática de todos los elementos coincidentes.

¿Es verdad? Sí, coinciden los conceptos.

Fuente que valida: MDN - Document.querySelector

6. ¿Qué es un evento?

Respuesta ChatGPT:

Un evento es una acción o suceso que ocurre en la página (clic, pulsación de tecla, carga, etc.) y que puede ser detectado por JavaScript para ejecutar código en respuesta.

Respuesta Copilot:

Un evento es cualquier interacción o cambio que ocurre en la página web, como un clic, el movimiento del mouse, la carga de la página, la escritura en un campo de texto, entre otros.

¿Es verdad? Sí, ambas definiciones coinciden.

Fuente que valida: MDN - Introducción a los eventos

7. ¿Qué es un listener?

Respuesta ChatGPT:

Un listener (o manejador de eventos) es una función que se asocia a un evento específico y se ejecuta cada vez que ese evento se dispara. Se agrega con addEventListener.

Respuesta Copilot:

Un listener (o event listener) es una función que "escucha" un evento específico en un elemento del DOM y ejecuta una acción cuando ese evento ocurre. Se agrega con el método addEventListener.

¿Es verdad? Sí, describen el mismo concepto.

Fuente que valida: MDN - addEventListener

Ejemplo de código usando un listener:

<button id="myBtn">Haz clic en mí</button>
<script>
  const myBtn = document.getElementById("myBtn");
  myBtn.addEventListener("click", function(event) {
    alert("¡Botón presionado!");
    console.log("Objeto de evento:", event);
  });
</script>

8. ¿Qué es un callback?

Respuesta ChatGPT:

Un callback es una función que se pasa como argumento a otra función y que se ejecuta posteriormente dentro de esa función. En el contexto de eventos, se usa como la función que se ejecutará cuando el evento ocurra.

Respuesta Copilot:

Un callback es una función que se pasa como argumento a otra función y se ejecuta después de que ocurre un evento o se completa una operación.

¿Es verdad? Sí, ambas definiciones son equivalentes.

Fuente que valida: MDN - Funciones callback

9. ¿Qué es un objeto de evento?

Respuesta ChatGPT:

El objeto de evento es un objeto que se pasa automáticamente a la función callback cuando se dispara un evento. Contiene información relacionada con el evento, como el tipo de evento, el elemento objetivo, posición del ratón, teclas presionadas, etc.

Respuesta Copilot:

Es un objeto que contiene información sobre un evento que ocurrió, como el tipo de evento, el elemento que lo disparó, las coordenadas del mouse, teclas presionadas, etc. Se pasa automáticamente como argumento a los callbacks de eventos.

¿Es verdad? Sí, es la misma definición.

Fuente que valida: MDN - Event
